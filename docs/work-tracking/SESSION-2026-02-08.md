# Session 2026-02-08 (Continuation): Matchmaker Fixes & Bracket Seeding

## Summary

Fixed multiple matchmaker tournament issues (bulk remove, PIN validation, registration flow) and added bracket seeding control for directors.

---

## Work Completed

### 7. Bulk Remove Registrations on Director Check-In Page
**Problem:** Director could bulk-select registrations and check them in, but had no way to bulk-remove them.

**Fix:**
- Modified `deleteMatchmakerRegistration` cloud function to accept `director_pin` as alternative auth (was player_id only)
- Added red "REMOVE SELECTED" button to batch action bar
- Added `removeSelected()` JS function with confirmation dialog and progress tracking
- Director can now select multiple registrations and delete them in bulk

**Files modified:**
- `functions/matchmaker.js` - Added director_pin auth path
- `public/pages/matchmaker-director.html` - CSS, HTML button, JS function

### 8. Fixed 5-Digit PIN Validation (Should Be 8-Digit)
**Problem:** Multiple pages still checked for 5-digit PINs instead of the 8-digit PINs the system uses.

**Fixes:**
- `public/pages/tournament-view.html` - `pin.length !== 5` → `pin.length !== 8`, updated error message
- `public/pages/stat-verification.html` - `pin.length < 5` → `pin.length !== 8`, updated error message
- `public/pages/matchmaker-register.html` - `pin.length < 4` → `pin.length !== 8`, updated error message

**Audit results:** All other PIN checks (x01-scorer 6-digit match PIN, dashboard 8-digit) were already correct.

### 9. Fixed Matchmaker Tournament Registration Flow
**Problem:** Clicking "REGISTER NOW" on tournament-view.html showed a generic registration modal that required selecting events. Matchmaker tournaments don't have events, so the form said "Welcome back!" (PIN worked) but silently failed to register.

**Fix:** When `matchmaker_enabled: true` on the tournament document, the REGISTER NOW button redirects to `matchmaker-register.html` instead of opening the generic modal.

**File modified:** `public/pages/tournament-view.html`

### 10. Bracket Seeding Control
**Problem:** Director could only random-shuffle or swap individual pairs in the bracket. No way to set a specific seed order.

**Implementation:**

**Backend** (`functions/tournaments/brackets.js`):
- `regenerateBracket` now accepts optional `seed_order` array of team IDs
- If provided, teams placed in exact order; missing teams appended at end
- Backwards-compatible: no seed_order = random shuffle

**Frontend** (`public/pages/matchmaker-director.html`):
- New "SEEDING ORDER" section below Round 1 Matchups preview
- Each team row shows: seed number, up/down arrow buttons, team name
- `renderSeedingList()` - Extracts teams from bracket, renders sortable list
- `moveSeed(index, direction)` - Swaps adjacent teams in seed order
- `applySeedOrder()` - Calls regenerateBracket with custom seed_order array
- `shuffleRandom()` - Calls regenerateBracket without seed_order (random)
- Old REGENERATE button hidden, replaced by the two new buttons
- Seeding section only visible when bracket exists and is not locked
